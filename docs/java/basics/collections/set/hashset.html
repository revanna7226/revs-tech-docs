üß© 1Ô∏è‚É£ What is HashSet in Java?#

‚û§ Definition:#

A HashSet is a collection in Java that:

  • Stores unique elements only (no duplicates)

  • Has no guaranteed order of elements

  • Provides constant time performance (O(1)) for basic operations like add(), remove(), and contains()

It‚Äôs part of the Java Collections Framework and implements the Set interface.

üì¶ Class Declaration (from JDK source)#

public class HashSet<E> extends AbstractSet<E>
        implements Set<E>, Cloneable, java.io.Serializable {
    private transient HashMap<E,Object> map;
    private static final Object PRESENT = new Object();
}

‚úÖ So internally, a HashSet is actually built on top of a HashMap!

üß† 2Ô∏è‚É£ Data Structure used in HashSet#

üìö Under the Hood:#

  • Internally, HashSet uses a HashMap to store its elements.

  • Each element of the set is stored as a key in the HashMap.

  • The map‚Äôs value is a dummy constant object (often called PRESENT).

This is how uniqueness is guaranteed: üëâ A HashMap cannot have duplicate keys, so neither can a HashSet.

‚öôÔ∏è 3Ô∏è‚É£ How HashSet Works Internally#

Let‚Äôs look step-by-step üëá

üèóÔ∏è Step 1: Creating a HashSet#

HashSet<String> set = new HashSet<>();

When you do this:

  • A new empty HashMap is created internally with default capacity 16 and load factor 0.75f.

üßæ Step 2: Adding Elements#

set.add("Apple");

Internally:

map.put("Apple", PRESENT);

If "Apple" is not already present as a key, it is inserted:

  • The hash code of "Apple" is computed.

  • Index = (hash & (n - 1)) ‚Üí finds the bucket.

  • If the bucket is empty, a new node is placed.

  • If the bucket already has elements (collision), it‚Äôs added as a linked node (or tree node in Java 8+).

If "Apple" already exists, the map ignores the insertion (so duplicates are not added).

üßæ Step 3: Checking Existence#

set.contains("Apple");

Internally:

map.containsKey("Apple");

üßæ Step 4: Removing Elements#

set.remove("Apple");

Internally:

map.remove("Apple");

‚úÖ Step 5: Iteration#

When you iterate a HashSet, it just iterates over the keys of the HashMap:

for (String s : set) {
    System.out.println(s);
}

Internally calls:

map.keySet().iterator();

üßÆ 5Ô∏è‚É£ Internal Working Mini Model (Pseudocode)#

Let‚Äôs look at the constructor and add() flow in simplified form:

public class HashSet<E> {
    private transient HashMap<E, Object> map;
    private static final Object PRESENT = new Object();

    public HashSet() {
        map = new HashMap<>();
    }

    public boolean add(E e) {
        return map.put(e, PRESENT) == null;
    }

    public boolean remove(Object o) {
        return map.remove(o) == PRESENT;
    }

    public boolean contains(Object o) {
        return map.containsKey(o);
    }

    public int size() {
        return map.size();
    }
}

üß† Key takeaway: üëâ HashSet is just a wrapper over HashMap where keys = elements and value = dummy object.

  • Load Factor: 0.75

When the number of elements exceeds 16 √ó 0.75 = 12, the set resizes its internal table (rehash).
